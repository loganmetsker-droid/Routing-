# v1.6: Production Hardening

## Objective

Make the dispatch product supportable and repeatable in production with stable migrations, reliable jobs, and observable operational behavior.

## Build Now

- Scenario-based test fixtures
- Seed data
- Migration discipline
- API validation
- Structured logging
- Background job reliability
- Restoreable backups
- Basic observability

## Suggested Hardening Targets

- Seed at least one realistic local delivery day with depot, drivers, vehicles, jobs, route plan, route run, and exceptions
- Add validation around planner commands, publish flow, assignment changes, and stop lifecycle transitions
- Add structured logs for publish, dispatch, reassignment, exception, and proof events
- Add retry and dead-letter handling for background work that affects dispatch state
- Test backup and restore paths with documented runbooks

## Operational Readiness Signals

- Deterministic migration process
- Rebuildable environments from seed data and scripts
- Reproducible scenario tests for planner and dispatch workflows
- Metrics and logs sufficient to debug planning and execution failures

## Exit Criteria

- Core workflows are covered by scenario fixtures and validation
- Backups and restore paths are documented and tested
- Logging and monitoring can explain failures in planner, dispatch, and realtime flows
- The product can be deployed repeatedly without ad hoc manual repair
