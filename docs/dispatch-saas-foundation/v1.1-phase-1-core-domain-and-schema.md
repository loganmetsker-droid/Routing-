# v1.1: Core Domain and Schema

## Objective

Introduce the minimum domain model needed to support planning, execution, assignment history, exceptions, auditability, and tenant boundaries.

## Core Entities

- `organizations`
- `users`
- `organization_memberships`
- `depots`
- `drivers`
- `vehicles`
- `jobs`
- `job_stops`
- `route_plans`
- `route_plan_stops`
- `route_runs`
- `route_run_stops`
- `route_assignments`
- `stop_events`
- `exceptions`
- `proof_artifacts`
- `audit_logs`

## Domain Rules

- Every business row belongs to an organization
- A job can have one or more stops
- `route_plans` are draft planning records
- `route_runs` are published execution records
- Execution stop rows are snapshots and must not mutate draft plan rows in place
- Assignments are explicit and historical
- Events and audits are append-only where practical

## Required Refactors

- Remove any implicit `route = execution` mapping
- Remove optimizer writes directly into frontend-shaped structures
- Replace stop arrays without IDs with stop entities that have identity
- Add publish state and assignment state before operational execution

## Backend Module Target

Map the current NestJS modules toward this split:

- `auth`: identity, sessions, role resolution, org scoping
- `organizations`: organization CRUD and settings
- `users`: memberships and profiles
- `depots`: depot CRUD and defaults
- `drivers`: availability and shift limits
- `vehicles`: capacities and readiness
- `jobs`: intake, stops, priority, service requirements
- `planning`: draft plan generation and planner commands
- `route-plans`: planned route storage and lock state
- `route-runs`: published route execution and actuals
- `assignments`: driver and vehicle assignment history
- `dispatch`: live board and day-of-operations actions
- `exceptions`: operational recovery queue
- `proofs`: note, signature, photo metadata
- `audit`: append-only action logs
- `realtime`: SSE or socket fanout from durable writes

## Exit Criteria

- Schema supports planning and execution as distinct concepts
- Organization scoping exists on all business tables
- Route assignment and publish semantics are explicit
- Stop identity, event history, and audit history are modeled first-class
