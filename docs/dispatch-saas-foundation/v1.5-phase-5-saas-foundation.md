# v1.5: SaaS Foundation

## Objective

Add the minimum SaaS controls required to operate multiple organizations safely without overbuilding commercial platform features too early.

## Build Now

- Tenant scoping on all business tables
- Organization creation
- Organization settings
- Role-based access
- Audit logging
- Environment config discipline
- Migrations
- Error monitoring
- Basic admin support tools

## Build Later

- Billing
- Subscription lifecycle
- Customer-facing API keys
- Webhooks
- Support impersonation

## Access Model

Support these roles first:

- `owner`
- `admin`
- `dispatcher`
- `viewer`
- `driver`

## Safety Rules

- Organization context must be resolved before business queries
- Read and write paths must enforce org scoping
- Material actions must write audit logs
- Realtime messages must originate from durable, org-scoped writes

## Exit Criteria

- Multi-organization separation exists in schema, services, and API guards
- Roles constrain what dispatchers, viewers, and drivers can do
- Audit logs exist for operationally important actions
- Environment and migration practices are fit for repeated deployment
