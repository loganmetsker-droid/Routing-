# v1.4: Optimizer Foundation

## Objective

Build a stable, deterministic heuristic optimizer that supports dispatch operations instead of chasing advanced research features.

## Build Now

- Deterministic batching
- Depot-aware route creation
- Basic multi-vehicle support
- Typed capacity support
- Service time support
- Shift limit support
- Lock-aware reoptimization
- Preserve-manual-edit logic

## Optimizer Rules

- Optimizer reads domain records, not raw frontend payloads
- Optimizer returns draft planning results, not operational execution state
- Reoptimization must preserve locked stops and manual edits where instructed
- Same input should produce materially consistent output

## Do Not Build Yet

- Predictive ETA machine learning
- Traffic intelligence
- Advanced metaheuristics
- Arbitrary pickup-and-delivery engine
- Cross-region enterprise optimization

## Internal Interface

The frontend should never call a raw optimize endpoint with arbitrary JSON. Use command endpoints that:

1. Load jobs, stops, settings, depot, drivers, and vehicles.
2. Run the internal optimizer module.
3. Persist draft route-plan results.
4. Return planner read models.

## Exit Criteria

- Draft generation is deterministic enough for dispatcher trust
- Capacity, service time, depot, and shift limits are enforced
- Reoptimization respects manual locks
- Optimizer remains a narrow internal module within the monolith
