# v1.2: Planning Workflow

## Objective

Make the planning workspace the core product screen and support credible dispatcher-controlled draft planning.

## Workflow Scope

- Jobs intake flow
- Unassigned jobs queue
- Planning workspace
- Draft route generation
- Manual route editing
- Driver assignment
- Vehicle assignment
- Publish flow
- Stop and route locking
- Reoptimization of unlocked work only

## Must Support

- Create routes from unassigned jobs
- Drag and reorder stops
- Move stops between routes
- Pin and lock stop sequence
- Mark job priority
- Assign vehicle
- Assign driver
- Publish a route plan into route runs

## API Shape

Prefer REST command endpoints:

- `GET /planner?serviceDate=YYYY-MM-DD`
- `POST /route-plans`
- `POST /route-plans/:id/generate-draft`
- `POST /route-plans/:id/reoptimize`
- `POST /route-plans/:id/publish`
- `POST /route-plans/:id/stops/reorder`
- `POST /route-plans/:id/stops/move`
- `POST /route-plans/:id/stops/lock`
- `POST /route-plans/:id/stops/unlock`
- `POST /route-plans/:id/routes/:routeGroupKey/assign-driver`
- `POST /route-plans/:id/routes/:routeGroupKey/assign-vehicle`

## Frontend Priority

Redesign the planning workspace first:

- Left rail: unassigned jobs
- Center: route cards and grouped draft routes
- Right rail: selected route detail and ordered stops
- Optional synchronized map pane

## Exit Criteria

- Dispatcher can generate a draft plan from unassigned jobs
- Dispatcher can manually edit route composition and stop order
- Driver and vehicle assignment happen before publish
- Locked work survives reoptimization
- Publish creates route runs instead of mutating draft rows in place
